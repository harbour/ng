 PACKDATA                      Enable packing of NEW EXE data segments
------------------------------------------------------------------------------

 Purpose  : Enable packing of NEW EXE data segments

 Syntax   : PACKDATA [nuThreshold]

 Default  : Disabled

 This command specifies that Blinker should pack data segments when creating
 NEW exe format executables (Windows and DOS Extended programs). The optional
 parameter is the maximum size for each segment before starting a new one.
 Data packing is disabled by default except when symbol table compression is
 enabled for CA-Clipper 5.x programs.

 The default threshold value, unless otherwise specified is 65535 bytes.
See Also: NOPACKDATA
