 PACKCODE                      Enable packing of NEW EXE code segments
------------------------------------------------------------------------------

 Purpose  : Enable packing of NEW EXE code segments

 Syntax   : PACKCODE [nuThreshold]

 Default  : Enabled (threshold=65500)

 This command specifies that Blinker should combine code segments when
 creating NEW exe format executables (Windows and DOS extended programs). The
 optional parameter is the maximum size for each segment before starting a
 new one. Code packing is enabled by default with a threshold of 65500 bytes,
 which is also the default if no parameter is specified.
