 1205: not enough memory to load procedure
------------------------------------------------------------------------------

 This message should only occur when using CA-Clipper Summer '87 with the
 dynamic method of overlay allocation (the default method for Summer '87).
 This message indicates that there is no more dynamic memory available in
 CA-Clipper's free pool to load the requested overlaid procedure.

 For a description of how to maximize the amount of dynamic memory available
 to CA-Clipper please refer to Chapter 8.
