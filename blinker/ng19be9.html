 NOPACKCODE                    Disable packing of NEW EXE code segments
------------------------------------------------------------------------------

 Purpose  : Disable packing of NEW EXE code segments

 Syntax   : NOPACKCODE

 Default  : Disabled

 This command specifies that Blinker should not pack code segments when
 creating NEW exe format executables (Windows and DOS extended programs.)

 Blinker defaults to packing code segments when creating NEW EXE format
 programs, as the NEW EXE format allows for only a limited number of segments
 (254).
