rlAnd()               AND together two RecordList's
------------------------------------------------------------------------------

Synopsis

    rlAnd(rl1, rl2) -> rlResult

Arguments

    rl1 & rl2 are the two RecordList's to be AND'd together.

Returns

    rl1 (AND'd with rl2).

Description

    rlAnd() AND's together two RecordList's so that only records contained
    in both RecordList's are contained in the result.

    rl2 is destroyed by this function.  The returned RecordList is actually
    rl1 (with its bit-maps properly AND'd).  This saves the the developer
    from having to explicitly destroy the AND'd recordList.

Example

    rl1 := rlNew(10)                && Set up rl1 with recs 5 & 7
    rlSet(rl1, 5)
    rlSet(rl1, 7)
    ? rlCount(rl1)                  && Will return 2

    rl2 := rlNew(10)
    rlSet(rl2, 5)
    rlSet(rl2, 9)
    ? rlCount(rl2)                  && Will return 2

    rl1 := rlAnd(rl1, rl2)          && AND them together
    ? rlCount(rl1)                  && Will return 1 (because only record
                                    && 5 was in both RecordList's)

    rlDestroy(rl1)                  && Now clean up (rl2 was destroyed
                                    && by the rlAnd() above)

Example

    rl1 := rlNew(10)
    rlSet(rl1, 5)
    rl2 := rlNew(10)
    rlSet(rl2, 5)
    rlSet(rl2, 7)
    ? rlCount(rl1), " records are in the first RecordList"
    ? rlCount(rl2), " records are in the second RecordList"
    rlResult = rlAnd(rl1, rl2)
    ? rlCount(rlResult), " records are in both RecordList's"

    ? (rlResult == rl1)             && Will return .T.
    ? (rlResult == rl2)             && Will return .F.
    rlDestroy(rlResult)

See Also

    rlOr(), rlNot()
