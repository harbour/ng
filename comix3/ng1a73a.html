cmKeyCount() includes deleted records!  Why?

cmKeyCount() counts the number of records in the index which meet the filter
condition.  The setting of SET DELETED is ignored.

If you want different behavior add ".and. !deleted()" to your condition
explicitly.  For example:

    cmFilter("<condition> .and. !deleted()")
    ? cmKeyCount()  && returns the number of keys in the index that 
                    && meet the condition (and therefore aren't deleted).
