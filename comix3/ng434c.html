cmKeyCount()          Return the number of keys in the index (filtered)
------------------------------------------------------------------------------

Synopsis

    cmKeyCount( [nOrder] | [cTag [, cBag]] ) -> nKeys

Arguments

    nOrder, cTag and cBag are optional parameters which specify the index.
    If omitted, the current master index will be used.

Returns

    cmKeyCount() returns the number of keys in the current which meet the
    filter condition (and fall within the current index scope if one is
    defined).

Description

    cmKeyCount() is similar to cmxKeyCount(), but returns the number of keys
    which meet the filtered condition in the current master index.
    (cmxKeyCount() returns the "raw" count regardless of the filter).
    
    If the index is not conditional (i.e., no FOR clause), this is the same
    as the number of records which meet the condition.

    If the index is conditional, this will be the number of records which
    meet the filter condition and meet the FOR condition of the index.

    cmKeyCount() will force Linear Optimization to be performed so that the
    count returned is fully accurate.

    cmKeyCount() obeys the current index scope (if any).  I.e., it will only
    count the number of keys for the current index which meet the condition
    _and_ fall within the current index scope.

Example

    index on age to age
    index on age ageFor for last = 'J'
    index on salary to salary
    use demo index age, ageFor, salary
    ...

    cmFilter("salary >= 75000")

    *   Get the number that meet the filter condition
    set order tag age
    ? cmxKeyCount(), " records match: " + dbFilter()

    *   Get the number that meet the filter condition and "last = 'J'"
    set order tag ageFor
    ? cmxKeyCount(), " records match: " + dbFilter() + " .and. " +;
            indexFilter()
