 BASE/2012    Open error (COPY FILE command)

     Default Behavior:  The program will continue without copying the
     file.

     Explanation:  In a COPY FILE command, the source file could not be
     opened.

     Action:  Check to make sure that the file exists.  If the file is
     not in the directory where the application runs, use SET DEFAULT or SET
     PATH to make the file accessible, or specify the full path name where
     the file can be found.

     See Also:  COPY FILE command
