 ert0030 Disallowed symbol '<symbol>' already loaded in .PLL '<prelinked
         library>'

     Explanation:  A symbol specified using the EXCLUDE option has
     already been loaded into a referenced .PLL.  The linker, therefore,
     could not avoid pulling in the module that defines the given symbol as
     intended.

     Action:  Use the EXCLUDE option for the symbol when building .PLLs.
