 C3008   Bad filename in #include

     Explanation:  You specified a #include directive with a file
     specification that is not a literal string enclosed in quotation marks
     (e.g., "Initial.ch").  In CA-Clipper, brackets (<>) are not supported.

     Action:  Enclose the literal filename in quotation marks and
     recompile.
