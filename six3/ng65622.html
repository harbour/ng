  Compact Indexes:

  Like FoxPro 2, The SIx Driver's SIXCDX and SIXNSX flavors create Compact
  indexes.  This means that the key data is stored in a compressed format
  resulting in a substantial size reduction in the index file.  The compact
  index size will be up to 90% smaller that the size of an equivalent .NTX
  index.  Since the amount of compression is dependent on many variables,
  including the number of unique keys in an index, the exact amount of
  compression is impossible to pre-determine.
