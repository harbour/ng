  FoxPro record-locking support:

  The SIx Driver's SIXCDX flavor uses FoxPro's record-locking scheme,
  instead of CA-Clipper's.  This allows your SIXCDX application to access
  data and index files concurrently with a FoxPro application, with both
  sides seeing each others record locks in a shared environment.

    NOTE: The SIXNSX and SIXNTX drivers both use CA-Clipper's record-
          locking scheme.
