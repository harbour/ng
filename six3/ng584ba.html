  SIX???/1022   Lock Required

  Problem:   An attempt was made to update a record in a shared database
             without first obtaining a lock.

  Solution:  Correct the program.  Obtain a record lock or file lock for
             the work area before attempting to update a record.  If shared
             access is not desired, use the EXCLUSIVE clause of the USE
             command to gain exclusive access to the database (.DBF) file.
