  BLIGPF.EXE:

  BLIGPF will scan a Blinker 3.x-created .MAP file for the symbol being
  referenced by the memory address information (Segment and Offset) displayed
  by a General Protection Fault (GPF) Error.  If found, the symbol and its
  associated information will be displayed.

  The usage syntax for BLIGPF is as follows:


  Usage:  BLIGPF [<MapFile>] [<SSSS:OOOO>]

  Where:  <MapFile> = Optional name of the Blinker 3.x MAP file to be
                      processed.  If no file name is specified, BLIGPF will
                      attempt to read it directly from the Blinker 3.x GPF
                      error message on the current video screen.  If this
                      fails, the first .MAP file in the current directory
                      (if any) will be assumed.

        <SSSS:OOOO> = Optional Segment and Offset values of the memory
                      address reported by the GPF error message.  (Example:
                      0005:01AE).  If no address is specified, BLIGPF will
                      attempt to read it directly from the Blinker 3.x GPF
                      error message on the current video screen.

                 /? = Displays this help screen.

  NOTE:  The Blinker 3.x MAP file MUST be created with MAP A or MAP S,A.


  NOTE:  This utility is also included on the Blinker 3.x distribution disk.
