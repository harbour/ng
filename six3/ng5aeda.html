  SIX???/680  Index Lock Timeout

  Problem:  Unable to get an index lock after the number of retries set
            set by the Sx_LockRetry() function.  If the LockRetry() has not
            been called, this error will never occur.

  Solution:  This is an informational warning.  In most cases, the
             appropriate action would be to display a message to the user
             indicating that the index is busy and cannot be locked.
See Also: Sx_ROXLock() Sx_ROXUnLock()
