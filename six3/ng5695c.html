  Error Codes:

  When an error is encountered with the SIx Driver, an error code is
  generated and passed to an error object in your error handler or you
  can call the API function Sx_Error() to check the error code for the
  last SIx Driver command or function.

  For the sake of compatibility and to reduce the amount of changes to your
  code we have used the same error code as DBFNTX wherever possible.

  NOTE: The error codes are listed as SIX???, where "???" will be "CDX"
        under the SIXCDX flavor of the SIx Driver, "NSX" under the SIXNSX
        flavor, and "NTX" under SIXNTX.
