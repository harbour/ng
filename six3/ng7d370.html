  Why do CA-Clipper applications using SIx 3.0 lock up on XT's?:

  To reduce the load size and improve performance, SIx 3.0's "C" code has
  been compiled using 286 optimizations.  This effectively prevents SIx 3.0
  applications from running on XT (8086) processors.
