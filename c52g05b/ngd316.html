 143 CHANGED: .OR.
--------------------------------------------------------------------------------

     In a departure from Summer '87, Clipper 5.0 shortcuts the
     evaluation of .OR. operands by evaluating the second condition only
     when the first condition is false (.F.).  If the first condition is
     false (.T.), the .OR. operation is true (.T.), and the second
     condition is never evaluated.
