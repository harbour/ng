 64  CHANGED: e:osCode
--------------------------------------------------------------------------------

     In Clipper 5.01, Error:osCode now properly reflects the DOS
     extended error code for file errors.  This allows proper
     distinction between errors which result from sharing violations
     (e.g., opening EXCLUSIVE when another process has already opened
     the file) and access violations (e.g., opening read/write when the
     file is marked read-only).
